# Data dictionary - Nestle diet-optimization demo

Generated by data/generate_sql.py. All tables live in `PK_NESTLE_DIET.DIET`.

Provenance: USDA FoodData Central SR Legacy (CC0, public domain) for the nutrition spine; Open Food Facts (ODbL) for branded products; Poore & Nemecek (2018, Science) processed by Our World in Data (CC BY 4.0) for CO2e/water; recipes, prices, and persona targets synthesized.

## commodity  (dim)

One row per traded commodity input. Master/dimension; the cost and price-volatility backbone for ingredients.

Primary key: `commodity_id`

| column | type | null | description |
|---|---|---|---|
| commodity_id | VARCHAR | NOT NULL | Commodity code (natural key), e.g. cocoa, coffee, wheat |
| commodity_name | VARCHAR | NOT NULL | Human-readable commodity name |
| unit | VARCHAR | NOT NULL | Quotation unit for the base price |
| base_price_usd_per_kg | FLOAT | NOT NULL | Reference price in USD per kg at index 1.00 (Jan 2025) |
| current_price_index | FLOAT | NOT NULL | Latest normalized price index (1.00 = Jan 2025) |
| current_price_usd_per_kg | FLOAT | NOT NULL | Latest price in USD per kg |
| peak_price_index | FLOAT | NOT NULL | Maximum price index observed over the history window |
| is_volatile | BOOLEAN | NOT NULL | TRUE for commodities with high recent price volatility (cocoa, coffee, olive oil) |

## commodity_price_history  (event)

One row per commodity per month. Time series of the monthly price index, Jan 2025 to Jun 2026, anchored to real commodity moves.

Primary key: `commodity_id, price_month`
  Foreign keys: `commodity_id` -> `commodity.commodity_id`

| column | type | null | description |
|---|---|---|---|
| commodity_id | VARCHAR | NOT NULL | Commodity code; FK to commodity |
| price_month | DATE | NOT NULL | First day of the month for this price observation |
| price_index | FLOAT | NOT NULL | Normalized price index (1.00 = Jan 2025) |
| price_usd_per_kg | FLOAT | NOT NULL | Price in USD per kg for the month |

## recipe_ingredient  (junction)

One row per ingredient used in a recipe (the bill of materials). Junction between recipe and ingredient with the quantity used.

Primary key: `recipe_id, ingredient_id`
  Foreign keys: `recipe_id` -> `recipe.recipe_id`; `ingredient_id` -> `ingredient.ingredient_id`

| column | type | null | description |
|---|---|---|---|
| recipe_id | NUMBER | NOT NULL | Recipe; FK to recipe |
| ingredient_id | NUMBER | NOT NULL | Ingredient; FK to ingredient |
| ingredient_name | VARCHAR | NOT NULL | Denormalized ingredient name for readability |
| quantity_g | FLOAT | NOT NULL | Quantity of the ingredient in this recipe, in grams |

## persona  (dim)

Primary key: `persona_id`

| column | type | null | description |
|---|---|---|---|
| persona_id | VARCHAR | NOT NULL | Persona code (natural key) |
| persona_name | VARCHAR | NOT NULL | Display name of the persona |
| description | VARCHAR | NOT NULL | Short description of the persona |
| sex | VARCHAR | NOT NULL | Biological sex used for nutrient reference values |
| age_years | NUMBER | NOT NULL | Age in years |

## dietary_restriction  (dim)

One row per dietary restriction. Reference for restriction rules that exclude ingredients carrying given allergen/diet flags.

Primary key: `restriction_id`

| column | type | null | description |
|---|---|---|---|
| restriction_id | VARCHAR | NOT NULL | Restriction code (natural key) |
| restriction_name | VARCHAR | NOT NULL | Display name of the restriction |
| description | VARCHAR | NOT NULL | What the restriction excludes |
| excluded_flags | VARCHAR | NOT NULL | Pipe-delimited ingredient flags excluded by this restriction |

## persona_restriction  (junction)

One row per persona per active dietary restriction. Junction linking personas to the restrictions they follow.

Primary key: `persona_id, restriction_id`
  Foreign keys: `persona_id` -> `persona.persona_id`; `restriction_id` -> `dietary_restriction.restriction_id`

| column | type | null | description |
|---|---|---|---|
| persona_id | VARCHAR | NOT NULL | Persona; FK to persona |
| restriction_id | VARCHAR | NOT NULL | Restriction; FK to dietary_restriction |

## persona_nutrient_target  (junction)

One row per persona per nutrient. The optimization constraints (floor/ceiling/range) the daily menu must satisfy.

Primary key: `persona_id, nutrient_code`
  Foreign keys: `persona_id` -> `persona.persona_id`

| column | type | null | description |
|---|---|---|---|
| persona_id | VARCHAR | NOT NULL | Persona; FK to persona |
| nutrient_code | VARCHAR | NOT NULL | Nutrient column code matching recipe/ingredient nutrition columns |
| nutrient_name | VARCHAR | NOT NULL | Human-readable nutrient name |
| unit | VARCHAR | NOT NULL | Unit of the min/max amounts |
| min_amount | FLOAT | NOT NULL | Minimum daily amount (floor); 0 if none |
| max_amount | FLOAT | null | Maximum daily amount (ceiling); null if none |
| direction | VARCHAR | NOT NULL | Constraint type: floor, ceiling, or range |
| priority | NUMBER | NOT NULL | Constraint priority 1 (highest) to 3 (lowest) |

## meal_slot  (dim)

One row per meal slot in a daily plan. Defines how many recipes of which meal type fill each slot in the menu optimization.

Primary key: `slot_id`

| column | type | null | description |
|---|---|---|---|
| slot_id | VARCHAR | NOT NULL | Slot code (natural key): breakfast, lunch, dinner, snack |
| slot_name | VARCHAR | NOT NULL | Display name of the slot |
| eligible_meal_type | VARCHAR | NOT NULL | Recipe meal_type eligible for this slot |
| min_recipes | NUMBER | NOT NULL | Minimum recipes assigned to this slot |
| max_recipes | NUMBER | NOT NULL | Maximum recipes assigned to this slot |
| slot_order | NUMBER | NOT NULL | Display order of the slot within a day |

## status_quo_menu  (fact)

One row per recipe in the persona's current 'favourite' daily menu. The fixed baseline whose cost spikes under a price shock; the Q3 re-solve protects against that spike.

Primary key: `slot_id, recipe_id`
  Foreign keys: `recipe_id` -> `recipe.recipe_id`

| column | type | null | description |
|---|---|---|---|
| slot_id | VARCHAR | NOT NULL | Meal slot this recipe fills |
| recipe_id | NUMBER | NOT NULL | Recipe; FK to recipe |
| recipe_name | VARCHAR | NOT NULL | Denormalized recipe name for readability |
| cost_usd | FLOAT | NOT NULL | Cost of the recipe at current prices |
| cost_usd_shock | FLOAT | NOT NULL | Cost of the recipe under the price-shock scenario |
